Функциональные требования:
Тема №17: "Генератор задания 23 ЕГЭ по информатике с вычислением ответа"

Пользователь взаимодействует с программой с помощью меню и клавиатуры.

В начальном меню пользователю выводится приветствие, название программы и предложение о выводе в окно консоли описания программы.
Описание программы содержит в себе информацию о том, что делает программа, с какими данными работает и что выводит в результате работы. 
Потом пользователю необходимо вводить значаения по ходу работы программы. Обо всех ограничениях ввода пользователя уведомляют заранее. При неверном вводе данных пользователю предлагается ввести значение снова. 

Пользователю предлагается ввести количество задач, которые требуется сгенерировать. Также пользователю дается возможность завершить программу вводом числа 0.
На ввод ожидается положительное число не более 100. При вводе числа 0 программа завершается.

Затем пользователю предлагается ввести количество шагов, с помощью которых можно попасть из начального числа в конечное. Наличие хотя бы 1 пути из начального числа в конечное не является обязательным к выполнению условием. Здесь и далее за путь принимается последовательность результатов выполнения команд исполнителя.
На ввод ожидается значение: 2 или 3.

После пользователю предлагается ввести количество точек, через которые обязательно должен проходить путь.
На ввод ожидается число в диапазоне от 0 до 3 включительно.

Также пользователю предлагается ввести количество точек, через которые нельзя проходить.
На ввод ожидается число в диапазоне от 0 до 3 включительно.

Далее консоль очищается и пользователю предлагается выбрать режим работы программы. Также пользователю предлагается вывести описание режимов.
Описание содержит в себе краткую характеристику и основную суть работы режимов.

Режимы работы:
1)	"Запись задачи в файл"
Этот режим служит для того, чтобы сгенерированные задачи и ответы к ним записывались в файл.
При выборе этого режима окно консоли очищается и пользователю предлагается выбрать файл в формате "txt" для сохранения данных.
Выбор файла:
1.	пользователь может указать путь к конкретному файлу;
2.	пользователь может выбрать для сохранения файл по умолчанию (текстовый файл созданный программой на рабочем столе).
На ввод ожидается число: 1 или 2. 
При выборе первого способа пользователю предлагается ввести путь к файлу.
Выполняется проверка введенных данных, т.е. проверяется существует ли файл. Если файла не существует пользователю сообщают об ошибке и возвращают на этап выбора файла.
Затем в файл сохраняются задачи и ответы.
2)	"Вопрос-ответ"
Этот режим служит для того, чтобы выводить сгенерированные задачи по очереди и реагировать на ответ пользователя.
При выборе этого режима работа идет по следующему алгоритму:
1.	окно консоли очищается и затем на него выводится сгенерированная задача. Программа ожидает ответ пользователя. На ввод ожидается положительное число или 0;
2.	на полученный ответ программа реагирует в зависимости от правильности ответа: "Верно" или "Ошибка";
3.	если были выведены не все задачи, то ход работы возвращается в первый пункт алгоритма и порядок действий повторяется. Если программа вывела все задачи, то алгоритм завершается.
После этого программа выводит статистику и оценку ответов пользователя.
Статистика состоит из количества всех задач и количества решенных задач.
Оценка ответов заключается в выводе основанном на процентном соотношении решенных задач:
0-30% - Решено слишком мало задач или ни одной задачи не решено вовсе.
31-60% - Решено небольшое количество задач, слишком много ошибок.
61-90% - Решено большинство задач, но есть ошибки.
91-100% - Решены все или почти все задачи.

После выбора и выполнения одного из режимов пользователь сможет выбрать: закрыть программу или перезапустить её.

Генерация задач
Задачи генерируются по шаблону:
//
Исполнитель преобразует число, записанное на экране.
У исполнителя есть три команды, которым присвоены номера:
1.Прибавить (число)(Умножить на (число))
2.Прибавить (число)(Умножить на (число))
3.Прибавить (число)(Умножить на (число))

Первая из них увеличивает число на экране на (число)(умножает число на экране на (число)), вторая увеличивает число на экране на (число)(умножает число на экране на (число)), третья — увеличивает число на экране на (число)(умножает число на экране на (число)).
Программа для исполнителя – это последовательность команд. Сколько существует таких программ, которые исходное число (начальное число) преобразуют в число (конечное число) и при этом траектория вычислений программы содержит число (числа) ... и не содержит число (числа) ...?
Траектория вычислений программы — это последовательность результатов выполнения всех команд программы.
//
Числа для команд сложения выбираются случайно в диапазоне от 1 до 5 включительно. Числа для команд умножения выбираются случайно в диапазоне от 2 до 5 включительно. Числа для команд одного арифметического действия не могут повторяться. 
Начальное и конечное числа выбираются случайно в диапазоне от 1 до 100 включительно, так чтобы начальное число было меньше конечного.
Числа, которые содержатся и не содержатся в траектории вычислений программы выбираются случайно из диапазона между начальным и конечным числом. Эти числа не могут повторяться.
Количество команд и количество точек, которые входят и не входят в траекторию программы, зависит от введенных пользователем значений.
